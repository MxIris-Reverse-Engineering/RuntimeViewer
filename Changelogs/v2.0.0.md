# v2.0.0

This is a major release representing a ground-up evolution of Runtime Viewer. The Objective-C runtime engine has been replaced with MachOObjCSection, Swift interface support has been added via MachOSwiftSection, and a new MCP integration enables LLM clients to inspect runtime information. The networking layer has been expanded to support Bonjour multi-device mirroring, code injection reconnection, and sandbox app inspection via local sockets.

**382 commits · 443 files changed · ~49,000 lines added since v1.3.0**

> **One-time manual install required**
>
> v2.0.0 is the first RuntimeViewer build to ship with Sparkle-based auto-updates.
> Users on `v1.x` or any earlier v2.x beta/RC must download and install this
> release manually — there is no auto-update path from a Sparkle-less build.
> Every release *after* v2.0.0 will be delivered through the in-app update flow.

## Highlights

- **Swift Interface Support** — View Swift type interfaces, type layouts, and enum layouts, powered by MachOSwiftSection
- **MachOObjCSection** — Replaced ClassDumpRuntime with a faster, more accurate ObjC introspection engine with category dumping support
- **MCP Integration** — LLM clients (e.g., Claude) can inspect runtime information via the Model Context Protocol
- **Auto-Update (Sparkle)** — Built-in Check for Updates with opt-in beta channel, EdDSA-signed archives, and an appcast served from GitHub Pages
- **Bonjour Multi-Device** — Connect to iOS/macOS devices on the local network with automatic engine mirroring, device icons, and section-based toolbar grouping
- **Code Injection Reconnection** — Injected processes survive app restarts via XPC endpoint registry and socket persistence
- **Export Interface Wizard** — Xcode-style multi-step wizard for exporting ObjC/Swift interfaces to files
- **Runtime Interface Transformers** — Customizable C type replacements, VTable offsets, and member address annotations
- **arm64e Code Injection** — Inject into arm64e system applications with helper service support

## New Features

### Swift Interface Support
- Generate Swift type interfaces from Mach-O binaries via MachOSwiftSection
- View Swift type layouts and enum layouts with memory offset display
- Swift section generation options (concurrency annotations, access control, etc.)
- VTable offset display for Swift methods
- Generic type indicator icon for Swift generic types

### MCP Integration (macOS 15+)
- MCP Server executable for runtime inspection via LLM clients
- In-process HTTP bridge architecture — app hosts bridge server, external MCP server connects as client
- Automatic port discovery with port file at `~/Library/Application Support/RuntimeViewer/mcp-bridge-port`
- Bridge starts automatically on app launch
- Tools: `ListImages`, `ListClasses`, `ListProtocols`, `GetInterface`, `GetMemberAddresses`, and more
- PascalCase tool naming for LLM-friendly discovery
- MCP status indicator in the main toolbar with popover showing connection state
- MCP Settings panel with port configuration and config copy buttons
- Debug/release MCP configuration separation
- Image management tools for loading/unloading frameworks

### Export Interface Wizard
- Xcode-style multi-step export wizard with configuration, selection, progress, and completion steps
- Export single or multiple runtime interfaces (ObjC/Swift) to files
- Coordinator-based architecture with proper navigation flow
- Handles C++ template filenames and special characters in export paths

### Runtime Interface Transformers
- Customizable C type replacement system for generated interfaces
- Independent ObjC and Swift transformer configurations
- Built-in transformer modules:
  - **C Type Replacement** — Replace raw C types with readable equivalents
  - **Swift Type Layout** — Display memory layout with field offsets
  - **Swift Enum Layout** — Display enum case memory layout
  - **VTable Offset** — Show virtual table offsets for Swift methods
  - **Member Address** — Annotate Swift function and computed property addresses
- Predefined token template presets with reorderable configuration in Settings UI
- Uppercase hex formatting for addresses and offsets

### Bonjour Multi-Device Support
- Connect to iOS and macOS devices on the local network via Bonjour
- Automatic Bonjour self-discovery filtering (skip own advertisements)
- iOS real device support with local network permission handling
- Multi-host engine mirroring — engines from remote peers appear in the local toolbar
- Per-source reconciliation prevents multi-host pushes from wiping peer state
- Device metadata transmitted via Bonjour TXT record (model, name, simulator flag)
- Device and app icons displayed in the source switch menu
- Section-based toolbar grouping by host (Local, Attached, each remote host)
- Proxy server architecture for re-advertising local engines to downstream peers
- Connection retry with configurable timeout for transient network failures

### Code Injection Reconnection
- **XPC Endpoint Registry** — Injected apps register their XPC listener endpoint with the Mach Service helper daemon; endpoints are keyed by PID with automatic cleanup via `DispatchSource` process monitoring
- **Direct XPC Reconnection** — Host app reconnects to injected processes on restart by fetching stored endpoints and sending `ClientReconnected` messages, bypassing the normal handshake
- **Socket Injection Persistence** — Injected socket endpoint records are persisted to disk (JSON in Application Support); on startup, alive processes are reconnected and dead entries pruned
- **Auto-Reconnect** — `RuntimeLocalSocketClientConnection` automatically retries connection when the underlying socket drops, with `pendingHandlers` replay on reconnect

### Sidebar Loading Progress
- `objectsWithProgress` AsyncThrowingStream API on `RuntimeEngine` for fine-grained loading feedback
- Determinate progress bar replaces the indeterminate spinner in the sidebar
- Phase-based progress descriptions (preparing, indexing Swift types, loading ObjC classes, etc.)
- Progress continuation bridging from `MachOSwiftSection` indexing events
- Remote progress push support — progress events are forwarded across Bonjour connections

### Helper Service
- Migrate to new `SMAppService` helper architecture
- Helper service version check with automatic reinstallation on version mismatch
- Only reinstall on explicit version mismatch, not on transient communication errors
- `KeepAlive` enabled in launchd plist for daemon stability
- Delay between unregister and reinstall to work around `SMAppService` bookkeeping timing
- Support for uninstalling legacy helper service from Settings

### Auto-Update (Sparkle)
- Integrated [Sparkle](https://sparkle-project.org) for secure, in-app automatic updates
- **Check for Updates…** menu item directly below **About**
- Updates settings page with "Check now", automatic-check toggle, check interval (hourly/daily/weekly), and opt-in pre-release (beta) channel
- EdDSA signature verification on every downloaded archive
- Appcast feed served from GitHub Pages (`docs/appcast.xml`); releases cut via annotated `v*` tags automatically update the feed
- Unified `ReleaseScript.sh` drives archive → notarize → sign → appcast → GitHub Release for both local runs and CI

### Settings
- Replace SettingsKit with native SwiftUI `NavigationSplitView` for the settings panel
- General, Helper Service, Transformer, MCP, Notification, and Updates settings pages
- Expanded transformer settings with per-module enable/disable and template presets
- MCP settings with port configuration, server path display, and config copy buttons

### Other Features
- **Filter Engine** — Filter runtime classes and protocols with fuzzy search support
- **Reorderable Bookmarks** — Drag-to-reorder bookmarks with dictionary-indexed storage in Application Support
- **Sandbox App Injection** — Inject into sandboxed apps via local TCP socket communication
- **Debug Log Export** — "Debug > Export Logs" menu for one-click OSLog export
- **arm64e Architecture** — Inject into arm64e applications (system apps); main app remains arm64, daemon and server frameworks compiled as arm64e
- **Connection Notifications** — System notifications on connect/disconnect with configurable options
- **Text View Skeleton** — Loading skeleton effect for smoother interface loading experience
- **Jump to Definition Icon** — Visual indicator for type references in the interface viewer
- **ObjC IMP Address Comments** — Method and property accessor IMP addresses shown in ObjC interface output
- **Expanded Field Offsets** — New generation option to show field offsets in expanded format

## Improvements

### Architecture
- Replace ClassDumpRuntime with MachOObjCSection for ObjC interface generation — significantly better performance, memory efficiency, and category support
- Extract section caching into `RuntimeObjCSectionFactory` and `RuntimeSwiftSectionFactory`
- Two-phase `RuntimeEngine` init (create, then connect) for cleaner lifecycle management
- Simplify coordinator communication with direct routing (remove coordinator delegates)
- Rename `AppState` → `DocumentState` for clarity in the document-based architecture
- Refactor communication layer with local socket, stdio, and direct TCP support
- `RuntimeSource` unified with role parameter instead of separate enum cases

### Logging
- Migrate from `Loggable` protocol to `@Loggable` macro with `#log(level, message)` syntax
- Structured logging throughout the communication and engine layers
- OSLog store integration for log export

### Code Quality

- Use `@MemberInit` macro for cleaner option/settings initialization
- Extract address formatting helpers for consistent hex display

### MCP

- MCP bridge scoped per-document (renamed from Window to Document provider)

### Build & CI
- Full macOS archive, export, and notarization release workflow with GitHub Actions
- Self-hosted runner support with configurable runner selection
- Import Apple intermediate certificates for self-hosted codesigning
- iOS Simulator build in release workflow
- Separate debug/release XPC service targets

## Known Issues

A comprehensive list of known issues is maintained in `Documentations/KnownIssues/2026-04-10-rc4-review-findings.md`. Key items for awareness:

- `RuntimeEngineManager` is not `@MainActor`-isolated; concurrent multi-device Bonjour connections have a theoretical thread-safety risk (M3.1)
- `objectsWithProgress` producer Task does not support cooperative cancellation when the consumer cancels (M6.1)
- Reconnection loop uses fixed 500ms retry without exponential backoff (M5.3)
- Proxy servers are created for mirrored engines, causing unnecessary resource usage in multi-host topologies (M3.2)
- Settings page selection is not persisted across window close/reopen (M8.2)

These are tracked for follow-up in a future patch release.

## Migration Notes

### From v1.x
- **Helper Service**: The helper daemon has been migrated to `SMAppService`. You must reinstall the helper from Settings > Helper Service after upgrading.
- **Bookmarks**: Bookmark storage has moved to Application Support. A one-time automatic migration runs on first launch.
- **MCP**: MCP integration is new in v2.0.0. To use it with an LLM client, configure the client to use the MCP server at the path shown in Settings > MCP.
