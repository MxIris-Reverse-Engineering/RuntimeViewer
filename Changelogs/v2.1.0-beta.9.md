# v2.1.0-beta.9

Ninth public preview of the **2.1** line. This is the releasable build of the
beta.8 payload: the beta.8 tag was created while preparing the release, but no
GitHub Release or Sparkle appcast entry was published from it. beta.9 carries
the same user-facing feature set plus the dependency pin needed for clean
release-channel package resolution.

This is a `beta` build -- only clients that opted in via
**Settings -> Updates -> Include pre-release versions** receive it.

---

## New Features

### Editable syntax themes

The Settings window now includes a **Theme** page. Users can duplicate the
built-in Xcode preset, edit token colors for light and dark appearances, choose
bold / italic styles, and adjust selection and background colors. Open
documents react to theme edits live, while custom presets stay persisted in
settings.

### Interface editor line numbers

The interface text view now has a line-number gutter, making generated headers
and Swift interfaces easier to scan and discuss.

### Sidebar scope filter

Runtime-object search gains a scope popover that can prefilter by object kind
and object properties before the text query runs. It supports grouped kind
selection and Any / Only / Exclude property rules, then expands matching
descendants so relevant nested objects are easier to reach.

---

## Improvements

- Theme rendering caches resolved colors and fonts, reducing per-token work on
  large interfaces.
- Theme observation is shared through a dependency-provided stream instead of
  installing duplicate observation chains per open document.
- The editor now applies the configured selection color, preserves explicit
  13-point font-size choices, and keeps deleted custom presets from trapping an
  edit sheet.
- Sidebar filter clearing restores the full list immediately, and filter
  controls only appear in the loaded image state.
- Sidebar row clicks now listen to user-proposed selection changes, so Back /
  Forward synchronization no longer pollutes selection history.

---

## Build & Dependencies

- `MachOSwiftSection` is pinned to revision
  `6097de4d5b56adf54ff269e2b3f9cd0e7424923e`, the first release-channel
  dependency revision in this line that exposes the `SwiftDeclarationRendering`
  product required by `RuntimeViewerCore`.
- Release CI now selects Xcode 26.6 and reads `MARKETING_VERSION` directly from
  the project file before archiving, avoiding an early Xcode workspace-load
  failure before xcodebuild logging starts.
