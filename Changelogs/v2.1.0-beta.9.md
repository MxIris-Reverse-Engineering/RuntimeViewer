# v2.1.0-beta.9

Ninth public preview of the **2.1** line. Where beta.8 was a release-pipeline
repair that carried the beta.7 payload forward unchanged, this build ships the
substantial feature work accumulated since then: multi-tab content navigation,
remap-based code injection for hardened daemons, and a batch of runtime-engine
and transformer fixes.

This is a `beta` build — only clients that opted in via
**Settings → Updates → Include pre-release versions** receive it.

---

## Multi-Tab Content Navigation

The main window now hosts multiple content tabs, so several runtime objects can
be open side by side instead of replacing one another in a single pane.

- **Window tab bar** drives content navigation directly from the title bar.
- **Open in New Tab** is available both from content-view type links and from
  sidebar rows, and the sidebar context-menu entry now carries a
  `tabbar.top.rectangle` icon to match the app's other iconized menus.
- **Navigation history menu** exposes the per-tab back/forward timeline, with the
  toolbar title sourced from a single source of truth.
- The **navigation timeline is now decoupled from tabs**, so history is tracked
  per navigation context rather than being tied to tab lifetime.
- Tabs are **keyed by identity**; **Close Tab / Close Window** and the dynamic
  **⌘W** binding are re-keyed against the current key window.
- Tab-close races and stale text/inspector rebuilds were hardened, the tab bar
  accessory is sized from its frame, and the title-bar accessory is installed
  after the window frame is set.
- Sidebar bookmark context-menu items were restored after the tab-bar rework.

---

## Code Injection

Injection into strict-seatbelt daemons now goes through a memory-remap path
instead of the previous hand-rolled runtime handoff.

- New `mach_vm_remap`-based injection dispatches strict-seatbelt daemons via a
  `file-map-executable` probe and remap wiring.
- The server exposes `runtime_viewer_server_start` for the remap loader, and the
  runtime handoff now defers to the MachInjector loader.

---

## Runtime Engine & Transformer

- Enum-layout **fixed-bit masks** are surfaced in the transformer, and
  `SwiftEnumLayout` is synced with the detailed enum-layout model.
- Swift transformer templates are deferred to `SemanticTransformer`.
- Restored **absolutePath serialization** for image bookmarks.
- Runtime image node state is encapsulated behind reactive outline selection
  predicates; connections are released during teardown, injected-engine
  connectivity is verified on attach, and the handshake-timeout error message is
  clearer.
