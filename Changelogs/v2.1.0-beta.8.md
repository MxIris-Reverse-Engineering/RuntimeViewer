# v2.1.0-beta.8

Eighth public preview of the **2.1** line. This build brings the new
user-editable **Theme** settings panel, line numbers in the interface editor,
and a sidebar scope filter for narrowing runtime objects by kind and property.
It also folds in the project-structure cleanup from PR #80, including stricter
dependency-injected singleton access and Xcode synchronized folders.

This is a `beta` build -- only clients that opted in via
**Settings -> Updates -> Include pre-release versions** receive it.

---

## New Features

### Editable syntax themes

The Settings window now includes a **Theme** page. Users can duplicate the
built-in Xcode preset, edit token colors for light and dark appearances, choose
bold / italic styles, and adjust selection and background colors. Open
documents react to theme edits live, while custom presets stay persisted in
settings.

The theme model is now data-driven: the previous hardcoded presentation theme
was replaced by persisted theme presets plus a resolved runtime theme used by
the content renderer. Long-time users keep their legacy font size through a
one-time migration.

### Interface editor line numbers

The interface text view now has a line-number gutter, making generated headers
and Swift interfaces easier to scan and discuss.

### Sidebar scope filter

Runtime-object search gains a scope popover that can prefilter by object kind
and object properties before the text query runs. It supports grouped kind
selection and Any / Only / Exclude property rules, then expands matching
descendants so relevant nested objects are easier to reach.

## Improvements

### Theme rendering is cheaper and more stable

Resolved theme colors and fonts are cached so large interfaces no longer rebuild
style objects per token. Theme change observation is shared through a
dependency-provided stream, avoiding duplicate observation chains across open
documents.

The editor now applies the configured selection color, preserves explicit
13-point font-size choices, keeps deleted custom presets from trapping an edit
sheet, and synthesizes italic rendering when the monospaced system font lacks an
installed italic face.

### Sidebar filtering feels more responsive

Clearing the runtime-object search field now restores the full list immediately
instead of waiting for the normal typing debounce. Filter controls only appear in
the loaded image state, the root sidebar uses autohiding scrollers, and the
scope popover keeps its sizing stable while disclosure groups expand or collapse.

### Navigation history is no longer polluted by programmatic selection

Sidebar row clicks now listen to user-proposed selection changes instead of all
selection notifications. Programmatic sidebar synchronization from Back /
Forward no longer pushes duplicate history entries or truncates the forward
stack.

## Architecture & Tooling

- Project singletons are now hidden behind `@Dependency` accessors, making
  substitution points explicit and preventing new direct `.shared` call sites.
- App startup responsibilities were split out of `AppDelegate` into focused
  lifecycle controllers for appearance, debug menu setup, helper-service version
  checks, updater startup, and toolbar overflow handling.
- The Simulator installer moved into its own Swift package target so the main
  app can open it without importing the full Settings UI module.
- The AppKit and UIKit Xcode projects now use synchronized folders to reduce
  project-file churn for added and removed files.
- The stale checked-in `UXKit.xcframework`, legacy launchd plist, and old app
  icon assets were removed.
- `RxAppKit` is updated to `0.5.3`, including the proposed-selection fix needed
  by the sidebar history change.
