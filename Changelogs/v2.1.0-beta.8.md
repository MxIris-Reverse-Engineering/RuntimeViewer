# v2.1.0-beta.8

Eighth public preview of the **2.1** line. This is a release-pipeline repair
build for the beta.7 payload: the app behavior is carried forward, while the
remote dependency graph is refreshed so clean CI builds no longer depend on
sibling local checkouts.

This is a `beta` build — only clients that opted in via
**Settings → Updates → Include pre-release versions** receive it.

---

## Build & Tooling

### Clean remote-dependency release builds restored

The release workflow now resolves the Swift runtime stack from published remote
tags when `USING_LOCAL_DEPENDENCIES` is off. A full GitHub Actions dry run passed
with the release workflow configuration, including the Catalyst helper archive,
the main macOS archive/export/notarization path, and the iOS Simulator artifact.

This keeps the beta channel releasable from a clean checkout instead of relying
on local development package paths.

---

## Dependencies

- **MachOSwiftSection** now resolves from `0.12.0-beta.6` instead of being
  pinned exactly to `0.12.0-beta.5`, picking up the published
  `SwiftDeclarationRendering` product required by `RuntimeViewerCore`.
- **RxUIKit** now resolves from `0.1.2`, which contains the UIKit outline-section
  bindings needed by the iOS target under remote dependency resolution.
- Resolved pins were refreshed for the distribution workspace, including
  **swift-demangling** `0.4.3` and **swift-semantic-string** `0.1.5`; stale
  transient pins that are no longer part of the resolved graph were removed.

---

## Carried Over From beta.7

All changes from [v2.1.0-beta.7](v2.1.0-beta.7.md) ship unchanged, including the
helper-daemon reinstall fix, proxy-reached runtime-engine fixes, centralized
communication layer, ObjC protocol ownership cleanup, nested sidebar
specialization identity preservation, popover sizing fixes, and the published
remote package rollout from that build.
